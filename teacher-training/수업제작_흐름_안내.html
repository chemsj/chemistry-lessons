교사 연수 · 시스템 안내

수업 자료는 이렇게 만들어집니다

코드 한 줄 쓰지 않고, 말로 요청하면 AI가 인터랙티브 화학 수업을 만듭니다. 이 문서는 그 제작 흐름과, 각 단계에서 시스템이 무엇을 하고 선생님이 무엇을 정해야 하는지를 한눈에 정리한 안내입니다.

📌 핵심 한 줄 — 선생님은 “무엇을 가르칠지”만 정하고, “어떻게 코드로 만들지”는 시스템이 맡습니다.
한눈에

이 시스템의 작동 원리

세 가지만 기억하면 됩니다.

💬

말로 요청합니다

“르 샤틀리에 원리 1차시 만들어 줘”처럼 평소 말로 지시합니다. 선생님은 코드를 몰라도 됩니다.

🔁

거시 ↔ 미시를 오갑니다

먼저 전체 설계도(거시)를 그리고, 슬라이드 한 장씩(미시) 만들며, 매번 전체 흐름으로 되돌아와 점검합니다.

📐

규칙이 품질을 지킵니다

교육 원칙·색 언어·디자인 규약·자동 점검이 항상 켜져 있어, 만드는 사람이 달라도 일관된 품질이 나옵니다.

단계별

각 단계 — 시스템과 선생님의 역할

단계마다 시스템이 알아서 하는 일과 선생님이 정해야 하는 일을 나란히 정리했습니다.

0

주제·범위 정하기

쉽게 — “무엇을, 누구에게, 얼마나” 말로 알려 주는 단계
시스템이 하는 일
  • 한꺼번에 묻지 않고 핵심부터 하나씩 되물어 범위를 좁힙니다.
  • 기존 단원·자료와 겹치는지 확인하고 알려 줍니다.
선생님이 정하는 것
  • 입력 — 단원/차시, 대상 학년·과목, 수업 시간(분).
  • 입력 — 꼭 다루고 싶은 개념·실험·맥락.
  • 선택 — 시스템이 제시한 보기 중 고르기(예: 앵커 실험).
1

설계도 그리기 (거시)

쉽게 — 슬라이드를 바로 안 만들고, 단원 전체 ‘밑그림’부터 그리는 단계
시스템이 하는 일
  • 학습목표·대상 오개념·이야기 흐름(도입→탐구→정리→평가)을 초안으로 정리.
  • 슬라이드 개요표(장마다 의도·활동유형·레이아웃)를 짭니다.
  • 설계 문서로 저장해 이후 제작·점검의 기준으로 삼습니다.
lesson-blueprint lesson-activity-types docs/specs/…-blueprint.md
선생님이 정하는 것
  • 확인·수정 — 학습목표가 의도와 맞는지.
  • 선택 — 겨냥할 오개념, 중심 앵커 반응.
  • 승인 — 슬라이드 구성(개수·순서)을 이대로 갈지.
2

빈 발표판 깔기

쉽게 — 내용 없는 빈 발표 틀(16:9)을 준비하는 단계
시스템이 하는 일
  • 완성된 빈 덱을 수업 폴더에 복사하고 제목·과목·단원명을 채웁니다.
  • 상단바·이전/다음·키보드 이동·줌·화면 맞춤은 이미 완성되어 자동 동작(건드리지 않음).
lesson-deck-shell 수업폴더/index.html
선생님이 정하는 것
  • 거의 없음 — 단원·소단원 이름만 확인하면 됩니다.
3

슬라이드 한 장 짓기 (미시 · 반복)

쉽게 — 한 장을, 결정 요소를 하나하나 정해 가며 완성하는 단계
시스템이 하는 일
  • 정해진 순서로 결정: ①이 장의 의도 → ②앞 장과 연결 → ③레이아웃(2단/단일) → ④활동 요소 → ⑤배치 → ⑥실제 내용.
  • 탐구 흐름을 지킴: 발문 → 시뮬/관찰 → (버튼) 정리. 정답을 먼저 안 보여 줍니다.
  • 섹션 한 개만 외과적으로 추가하고 자동 점검(린트)을 돌립니다.
slide-compose lesson-slide-2col / single lesson-simulator-particle lesson-graph lesson-comprehension-check lesson-reveal-gate
선생님이 정하는 것
  • 선택 — 레이아웃, 어떤 활동(입자 시뮬·그래프·O/X 체크·사진 등).
  • 확인 — 발문 문구, 수치·반응식 계수, 실측 앵커가 맞는지.
  • 지시 — “이 장은 슬라이더를 넣자”처럼 요소를 더하거나 빼기.
  • 확인 — 완성된 한 장을 보고 다음으로 넘어갈지.
4

전체 흐름 점검 (거시 복귀 · 반복)

쉽게 — 한 장 만들 때마다 ‘전체적으로’ 자연스러운지 되돌아보는 단계
시스템이 하는 일
  • 이야기 흐름·앞뒤 연결·중복/공백·난이도 곡선·시각 언어 일관성을 점검.
  • 어긋나면 짧게 제안하고, 합의되면 설계도를 고쳐 해당 장만 다시 만듭니다.
flow-check
선생님이 정하는 것
  • 판단 — 제안(장 추가·교체·순서 변경)에 동의할지.
  • 이후 3번으로 돌아가 다음 슬라이드를 만듭니다(루프).
5

완료 전 검증

쉽게 — “다 됐다”고 말하기 전에 실제로 확인하는 단계
시스템이 하는 일
  • 자동 점검 오류 0, 화면 세로 넘침 0, 콘솔 오류 0 확인.
  • 시뮬 표시값 = 입자 수 = 그래프 = 계산값이 일치하는지 대조.
  • 글자 대비·반응식 계수·표기까지 확인한 뒤에야 완료로 보고.
lesson-verify scripts/lint-lesson.mjs
선생님이 정하는 것
  • 확인 — 미리보기로 직접 보고 최종 승인.
  • 지시 — 고칠 곳이 있으면 그 장만 다시 요청.
바탕

모든 단계에 항상 켜져 있는 규약

아래 두 가지는 특정 단계가 아니라 처음부터 끝까지 바탕에 깔려 품질과 일관성을 지킵니다.

① 교육 원칙

chem-pedagogy-guidelines
  • 탐구 우선 — 정답을 먼저 보여주지 않고, 관찰 뒤 버튼으로 정리.
  • 실측 앵커 — 시뮬 수치를 실제 측정값에 맞춰 “안 맞는다” 오개념 차단.
  • 학생이 배운 표기만 — 농도는 대괄호 [ ], 미학습 명칭은 이름 없이 설명.
  • 한 화면·한 핵심 — 한 슬라이드에 한 가지, 세로 넘침 0.
흡열 = 빨강 발열 = 파랑 색 언어는 모든 단원 고정

② 디자인 시스템 · 자동 점검

references/design-system.md · scripts/
  • 단일 진실 원천 — 색·폰트·클래스를 한 곳에서 정의(색은 변수로만).
  • 프레임워크 자동 — 네비·줌·발표 기능은 손대지 않아도 동작.
  • 자동 점검(린트) — 문법·중복·연결 오류를 기계가 잡아냄.
  • 외과적 변경 — 고칠 땐 그 슬라이드만, 전체를 다시 쓰지 않음.
