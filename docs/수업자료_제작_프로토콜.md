# 수업 자료 제작 프로토콜 (HTML 인터랙티브 수업)

단원마다 **단일 HTML 인터랙티브 수업 자료**를 일관된 품질로 만들기 위한 표준 절차입니다.
탐구 중심(예측→관찰→설명), 시뮬레이션·실험·데이터 기반 학습, 실제 사진 첨부를 기본으로 합니다.

- **구현 레퍼런스(기술 사용법)**: `README_수업제작_가이드.md` (템플릿 컴포넌트·헬퍼 함수)
- **완성 예시**: `equilibrium_lesson_16x9.html`(1세대), `concentration_pressure_lesson.html`(2세대 최신)
- **틀**: `lesson_template.html` 을 복사해서 시작

> 이 문서는 *무엇을 어떤 순서로* 만드는지(프로세스), README는 *어떻게 코딩*하는지(구현)를 다룹니다.

## 0. 산출물·폴더 규칙

```
단원명_폴더/
├─ index.html            ← 수업 자료 본체(템플릿 복제)
├─ img/                  ← 실제 사진·그림
│   ├─ 01_도입.jpg
│   ├─ 02_실험_전.jpg
│   └─ ...
├─ data/                 ← (선택) 탐구용 데이터(csv 등)
└─ 출처.md               ← 사진·데이터 출처/저작권 기록
```
- 파일명은 `index.html` 로 두면 웹 배포 주소가 깔끔합니다.
- 단원 번호를 접두어로(`02_농도압력_평형이동/`) 정렬되게.

### 0-1. 작업 핸드오프 (다른 기기/세션에서 이어받기)
- 워크스페이스 루트에 **`HANDOFF_작업로그.md`** 를 두고 항상 최신으로 유지한다. 다른 기기에서 새 세션을 시작하면 **이 파일을 가장 먼저 읽는다.**
- 포함 항목: ① 폴더·파일 명세(작명 규칙) ② 단원별 진행 현황 표 ③ 단원별 핵심 결정·데이터 ④ 미리보기/검증 워크플로 ⑤ TODO.
- 새 단원/큰 변경을 마칠 때마다 HANDOFF의 현황 표와 TODO, "최종 업데이트" 날짜를 갱신한다.

## 2. 제작 단계 (Analyze → Design → Prepare → Build → Verify → Publish)

### 2-1. 분석 (Analyze)
교과서·성취기준을 읽고 정리:
- 핵심 개념 / 선수 개념 / **대표 오개념**
- 탐구 요소(관찰·변인·예측·설명)와 다룰 **데이터**
- 안전 유의(유독·위험 실험은 사진·영상으로) — §5
→ 산출: **단원 분석 메모**(개념 구조 + 차시 범위)

### 2-2. 설계 (Design)
**차시 흐름**을 슬라이드 맵으로:

| 단계 | 슬라이드 | 핵심 |
|---|---|---|
| 도입 | 발문(single) | 선수개념 복습 + 학습목표 + 흥미 유발(사진/현상) |
| 전개 | 탐구 1~3 (2단) | §3 탐구 활동 유형에서 선택 (실험·시뮬·데이터) |
| 정리 | 종합(single) | 표·마인드맵으로 개념 통합 |
| 평가 | 형성평가(2단) | 오개념 점검 3문항 |
| 마무리 | 마인드맵(single) | 한눈에 정리 + 다음 차시 예고 |

원칙:
- **1슬라이드 1핵심**. 왼쪽=질문/정리, 오른쪽=탐구(시뮬/사진/데이터).
- 각 전개 슬라이드는 **예측 → 조작/관찰 → (버튼)핵심 정리(게이팅)** 흐름.

### 2-3. 자료 준비 (Prepare)
- 사진 수집·정리(§5), 데이터 표 정리, 시뮬레이션 사양(반응식·변인·평형값) 확정.
- 시뮬 상수는 **작은 노드 스크립트로 평형값/이동 방향을 미리 검증**(README 17장).

### 2-4. 제작 (Build)
- `lesson_template.html` 복제 → `index.html`
- 상단바(과목·단원), 슬라이드 채우기, 사진·시뮬 삽입.
- 입자 시뮬은 템플릿의 **예시 슬라이드(`initDemo`)를 복사**해 우리 반응으로 변형.

### 2-5. 검증 (Verify) — README 17장
- `node --check`(JS 문법), 세로 넘침 0, 콘솔 오류 0, 시뮬 수치 대조, 빔프로젝터 가독성.

### 2-6. 배포 (Publish)
- GitHub Pages / Netlify Drop 등 정적 호스팅 → 링크 공유(README 별도 안내).

## 3. 탐구 활동 유형 카탈로그

상황에 맞게 **2~4가지를 조합**합니다. 각 유형의 구현은 README 컴포넌트를 사용.

### A. 실험 관찰 (사진·영상 + 결과 입력)
- **언제**: 실제 실험이 핵심일 때(색 변화, 침전, 기체 발생 등). 위험·유독 실험은 **사진/영상으로 대체**.
- **구성**: 왼쪽 발문·안전 → 오른쪽 **실험 사진(전/후)** + 관찰 결과 선택/입력 → 핵심 정리.
- **구현**: §5 사진 컴포넌트 + `reveal()` 게이팅.

### B. 미시 입자 시뮬레이션 (보이지 않는 동적 변화)
- **언제**: 분자·이온 수준의 변화(동적 평형, 반응 진행, 상태 변화, 확산 등).
- **구성**: 입자 박스 + 시간 그래프 + 수치 칩 + 속도/일시정지/초기화.
- **구현**: README 12·14장 헬퍼(`pSeed·drawMolecule·reconcile·lineGraph`), 템플릿 예시 복사.

### C. 데이터 분석 / 그래프 (표 → 그래프 → 규칙)
- **언제**: 측정값에서 규칙·관계를 *추론*하게 할 때(농도비, 속도, 주기성 등).
- **구성**: 표(`.tbl`)에서 값 읽기 → 계산기/그래프로 규칙 확인 → 결론.
- **구현**: `.tbl`(셀 강조 `hi`), 계산기 패턴, `lineGraph`. 원자료는 `data/`에 보관.

### D. 예측-관찰-설명 (POE: Predict–Observe–Explain)
- **언제**: 오개념 교정·개념 변화가 목표일 때.
- **구성**: ① 예측 선택(형성평가식) → ② 시뮬/사진으로 관찰 → ③ 핵심 정리에서 설명.
- **구현**: `check()`(예측) + 시뮬/사진 + `reveal()`.

### E. 가상 실험 (변인 조작 슬라이더)
- **언제**: 변인-결과 관계 탐색(온도·압력·농도·부피 등).
- **구성**: 슬라이더로 변인 조작 → 결과(색·그래프·수치) 실시간 반응 → 경향 발견.
- **구현**: `.slider` + `data-init` 갱신 함수 + 수치 칩/그래프.

### F. 계산기 · 식 빌더
- **언제**: 식 세우기·정량 계산(평형 상수, 농도, pH 등).
- **구성**: 입력 → 계산 결과, 또는 계수 조절로 식 완성.
- **구현**: 계산기 패턴(README 5-B), `.frac`/`keq` 분수 표기.

### G. 형성평가 / 오개념 점검
- **언제**: 모든 단원 마무리 직전.
- **구성**: 3문항 내외, 대표 오개념을 정조준, 즉시 피드백+점수.
- **구현**: `oxbox`/`check()`/`scoreband`.

> **탐구 설계 팁**: 정답(정리)은 처음부터 보여주지 말고 **조작·관찰 후 버튼으로 펼치기**(게이팅). "발문 → 직접 조작 → 정리" 순서를 강제하면 탐구가 살아납니다.

## 5. 실제 사진·미디어 첨부 (중요)

### 5-1. 어디서 구하나 (출처·저작권)
| 출처 | 비고 |
|---|---|
| **직접 촬영** | 가장 안전·권장. 실험 장면·시약·결과를 직접 찍기 |
| **위키미디어 공용**(commons.wikimedia.org) | CC 라이선스. 출처·라이선스 표기 필요 |
| **공공누리**(공공기관, data.go.kr 등) | 공공저작물, 유형별 이용조건 확인 |
| **NASA/NOAA/USGS** | 대부분 퍼블릭 도메인(우주·지구·기상) |
| 교과서·문제집 사진 | **저작권 보호**. 수업 내 제한적 인용만, 가급적 **대체**(직접촬영/CC) |

- 모든 사진은 **`출처.md`에 출처·라이선스 기록**, 화면에는 `.figcap .src`로 출처 표기.
- **안전**: 유독(예: NO₂)·폭발성·고온 실험은 **직접 시연 대신 사진/영상**으로(교과서도 동일).

### 5-2. 폴더에 넣고 삽입 (권장: 파일 동봉)
사진을 `img/` 에 넣고:
```html
<figure class="figure">
  <img src="img/02_황산구리_가열전.jpg" alt="가열 전 파란색 황산 구리(II) 오수화물">
  <figcaption class="figcap">가열 전 — 파란색 CuSO₄·5H₂O
    <span class="src">출처: 직접 촬영 / ○○ (CC BY)</span>
  </figcaption>
</figure>
```
사진 2장 비교(전/후):
```html
<div class="photogrid">
  <figure class="figure"><img src="img/before.jpg" alt="전"><figcaption class="figcap">가열 전</figcaption></figure>
  <figure class="figure"><img src="img/after.jpg"  alt="후"><figcaption class="figcap">가열 후</figcaption></figure>
</div>
```
> `.figure`/`.photogrid`/`.figcap` 컴포넌트는 템플릿에 포함되어 있습니다. 전체가 들어가도록 사진은 **한 변 1000~1600px, 200KB 내외**로 줄여서 사용.

### 5-3. 단일 파일로 유지하고 싶을 때 (base64 내장)
사진까지 HTML 한 파일에 담아 공유가 편하게 하려면 data URI로 변환:
```bash
# 한 장을 data URI로 출력 (mac/리눅스)
printf 'data:image/jpeg;base64,'; base64 -i img/before.jpg
```
출력을 `<img src="data:image/jpeg;base64,...">` 에 붙여넣기.
- 장점: **파일 하나로 끝**(드라이브/메신저 공유 편함, 오프라인 OK).
- 단점: 용량 ~33% 증가. 사진이 많거나 크면 **§5-2(파일 동봉)** 가 낫습니다.

### 5-4. 외부 URL (비권장)
`<img src="https://...">` 는 인터넷 필요 + 링크가 깨질 수 있어 수업용으로는 비권장.

### 5-5. 영상
유튜브/실험 영상은 **링크 버튼**으로(임베드보다 안정적):
```html
<a class="btn ghost sm" href="https://..." target="_blank">▶ 실험 영상 보기</a>
```

## 7. 품질 체크리스트 (배포 전 한 장 요약)

- [ ] 학습목표가 도입에 함축적으로 제시됐는가

**시각·인터랙션 (이번 차시 정립)**
