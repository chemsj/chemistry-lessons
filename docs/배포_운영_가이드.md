## 1. 가장 자주 하는 일 — 고치고 올리기 (3줄)
파일을 수정한 뒤, 작업 폴더에서:
```bash
cd "…/Lesson Prep"
git add -A
git commit -m "무엇을 바꿨는지 한 줄(한국어)"
git push
```
- 1~2분 뒤 공개 주소에 자동 반영됨(GitHub Pages가 다시 빌드).
- 브라우저에 옛 화면이 보이면 **새로고침(Cmd+Shift+R)** — 캐시 때문.

---

## 2. 새 수업(소단원) 추가하기
예: 「화학의 언어」에 '화학식량과 몰' 수업을 새로 만든다고 하면.

1. **수업 폴더 + 파일 만들기** (대단원 폴더 안에 `NN-슬러그/index.html`)
   ```bash
   mkdir -p "chem-1-language/02-mole"
   cp "docs/lesson_template.html" "chem-1-language/02-mole/index.html"
   ```
   - `NN` = 두 자리 순번, 슬러그 = 영문 소문자-하이픈. 사진 쓰면 `mkdir chem-1-language/02-mole/img`.
   - 템플릿을 복제해 슬라이드를 채운다(제작은 README·프로토콜 참고).

2. **대단원 허브에 카드 한 줄 추가** — `chem-1-language/index.html` 의 `const lessons = [...]` 배열에 추가:
   ```js
   { "title":"화학식량과 몰", "file":"02-mole/", "desc":"몰과 아보가드로 수", "tag":"" }
   ```
   - `file` = 수업 폴더(끝에 `/`). 비워두면(`null`) **"준비 중"** 비활성 카드가 됨.
   - `tag` = `"신규"` 같은 배지(없으면 `""`).
   - 카드 순번(01,02…)은 배열 순서대로 자동.

3. **저장 → §1의 3줄로 push.** 끝.

## 3. 대단원/과목 노출 바꾸기 (루트 허브)
루트 `index.html` 의 `const subjects = [...]` 만 수정.
- 대단원에 자료가 생기면 그 칸의 `desc`(한 줄 설명)·`tag:"자료 있음"` 채우기.
- 대단원 카드 `file`은 대단원 폴더(`chem-1-language/` 등) — 바꿀 일 거의 없음.
- 새 과목/대단원을 늘릴 일이 생기면 배열에 한 칸 추가하고, 같은 이름의 폴더+허브 `index.html`을 만든다(기존 대단원 허브 복제해서 제목·`lessons`만 교체).

---

## 4. "준비 중" 대단원을 실제 자료로 바꾸기
지금 `chem-1·2·4`, `me-1·2·3·4` 대단원은 허브만 있고 "준비 중" 상태.
- §2처럼 소단원을 만들고 그 대단원 허브 `lessons`에 카드를 채우면 됨.
- 루트 허브에서 그 대단원에 `tag:"자료 있음"`·`desc`를 달아주면 학생이 자료 있는 단원을 한눈에 구분.

## 5. 올리기 전 점검 (수업 데크)
새/수정 데크는 push 전에 로컬에서 확인:
1. **로컬 미리보기**: 작업 폴더에서 `python3 -m http.server 8770` → 브라우저 `http://localhost:8770/` (또는 에디터 미리보기 `lesson-preview`).
2. 클릭 동선: 허브 → 대단원 → 수업이 **404 없이** 열리는지.
3. 데크 자체: **세로 넘침 0, 콘솔 오류 0**, 슬라이더·토글·초기화 동작(상세 체크는 README §17·프로토콜 체크리스트).
4. 화학식·계수·첨자·단위, 한국어 줄바꿈.

## 6. 주의 (하면 안 되는 것 / 함정)
- **교과서 PDF·`_archive-화학2(2015)/`는 git에 올리지 않는다** — 이미 `.gitignore`로 막혀 있으니 그대로 두기(저작권·용량·100MB 파일 제한).
- 사진은 **용량을 줄여서**(가로 ~1600px, 수백 KB) `img/`에 넣기 — GitHub 저장소가 무거워지지 않게.
- 폴더/파일명은 **영문 슬러그**(한글·공백 지양) — URL이 깔끔하고 안전.
- 루트·대단원 허브의 `file` 경로와 **실제 폴더명이 정확히 일치**해야 함(대소문자·하이픈). 안 맞으면 카드 클릭 시 404.
- 커밋 메시지는 한국어로, 무엇을 바꿨는지 한 줄.
