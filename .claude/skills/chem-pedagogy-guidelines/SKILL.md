# 화학 수업 제작 원칙 (상시 적용)

수업 자료를 **설계·제작·검토하는 모든 단계**에서 이 원칙을 기준선으로 삼는다. blueprint·slide-compose·flow-check가 공통으로 참조한다. 구체적 코드·토큰은 [design-system](../references/design-system.md)을 본다.

## 핵심 원칙 (요약)
1. **탐구 우선** — 정답을 먼저 보여주지 않는다. 흐름은 `발문(질문) → 시뮬레이션/관찰 → (버튼으로) 핵심 정리`. 핵심 정리는 `revealbtn`으로 활동 뒤 펼친다.
2. **과학적 정확성 = 실측 앵커** — 시뮬 제작 전 핵심 수치를 검증하고, 가능하면 **실측값을 앵커**로 쓴다(예: CoCl₂ `K(25℃)=1.7×10⁻³`, 산소해리 Hill식, van 't Hoff). 표시 농도(mol/L)를 식에 넣으면 화면 Q가 나오도록 모형을 구성해 "값이 안 맞는다"는 오개념을 차단. 단순화로 임의 단위를 쓰면 **"상댓값"임을 슬라이드에 명시**하고 경향·방향은 실제와 일치시킨다. 출처는 figcap `.src` 또는 단원 `출처.md`에.
3. **오개념 인지** — 흔한 오개념을 겨냥해 발문·형성평가를 설계. 텍스트("소모/감소")와 막대·시뮬 방향이 **반드시 일치**해야 한다(예: 법랑질 용해 시 PO₄³⁻·OH⁻는 H⁺에 소모되어 감소).
4. **표기는 학생이 배운 것만** — 농도는 대괄호 `[ ]`. 교육과정 밖 명명 법칙(헨리 법칙 등)은 **이름 없이** 평형·르샤틀리에로 설명. 근거는 교사용 `출처.md`에만.
5. **서술 대신 칩 흐름** — 핵심 정리·마무리는 긴 문장 대신 `아이콘+조건 → 중간 → 색 칩` (`.flowrow`)으로 인과 사슬을 한눈에. 흡열=빨강·발열=파랑 색 언어 고정.
6. **최소 인지 부하 / 한 화면** — 한 슬라이드에 한 가지 핵심. 콘텐츠가 720px를 넘기면 줄인다(세로 넘침 0이 목표). 군더더기 주석("[ ]는 몰 농도" 등)은 생략.
7. **학생이 직접 조작** — 시뮬에는 속도·일시정지·초기화·변인 슬라이더를 붙여 능동적으로 만든다. 데이터 탐구는 "측정하듯" 한 점씩 누적(자동 시작 금지).

## 제작 규율 (karpathy 가이드라인 응용)
- **성공 기준을 먼저 정의하고 검증될 때까지 반복** — 슬라이드마다 "학생이 무엇을 얻는가(학습 의도)"를 먼저 정하고, 완성 후 그 의도에 비춰 점검한다.
- **최소한만, 추측 금지** — 요청·학습목표에 필요한 것만 만든다. 슬라이드·요소를 과설계하지 않는다.
- **외과적 변경** — 기존 수업을 고칠 때 그 슬라이드/요소만 건드리고 전체를 재작성하지 않는다.
- **근거 우선** — "동작한다/맞다"고 단정하지 말고 린트·계산·미리보기로 확인한 뒤 보고한다(불일치는 그대로 보고).
- **불확실성은 드러낸다** — 수치·표기·계수가 불확실하면 숨기지 말고 표시하고 검증한다.

## 자가 점검 (완성 전)
- [ ] 학습 의도가 분명하고, 슬라이드가 그것을 달성하는가?
- [ ] 화학 반응식 **계수**가 제목·표·마인드맵 등 모든 곳에서 일치하는가?
- [ ] 시뮬 표시값 = 캔버스 입자 수 = 그래프 = Q 계산이 **같은 값**인가?
- [ ] 텍스트 설명 방향과 막대·시뮬 방향이 일치하는가?
- [ ] 흡열=빨강·발열=파랑 색 언어를 지켰는가?
- [ ] 세로 넘침 0, 콘솔 오류 0인가? (→ [verify 단계](../lesson-verify/SKILL.md) / `scripts/lint-lesson.mjs`)
