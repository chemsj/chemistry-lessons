# course-onboarding — 새 과목 시작 (성취기준 확정 + 교과서 인덱스)

새 과목을 시작할 때 **단원 제작([[lesson-blueprint]]) 전에 한 번** 밟는 준비 절차. 핵심 원칙: **"웹보다 폴더 먼저"** — 출판사 공통자료에 공식 교육과정이 다 들어 있다. 산출물은 `docs/specs/<과목>-curriculum.md` 와 `docs/specs/<과목>-textbook-index.md`.

## 사전 조건
- 교과서·교육과정 PDF가 `교과서파일/<과목>/<출판사>/` 에 있음(`.gitignore`, 로컬 전용). 학교 채택본 확인(★).

## 1단계 — 성취기준 확정 (공식 [별책9])
출판사 **공통자료**의 `[별책9] 과학과 교육과정.pdf`는 텍스트 문서라 `pdftotext`가 깔끔하다.
```bash
pdftotext "<공통자료 경로>/[별책9] 과학과 교육과정.pdf" /tmp/b9.txt
grep -nE '<과목코드>[0-9]{2}-[0-9]{2}' /tmp/b9.txt    # 예: 12물에01-01 → 성취기준 위치
sed -n '<시작>,<끝>p' /tmp/b9.txt                      # 그 구간(성취기준+탐구활동+해설) 추출
```
추출해 정리할 것: **성취기준 문구 · `<탐구 활동>`(공식 권장 실험) · 성취기준 해설(범위 제한) · 적용 시 고려 사항(연계·디지털 소양·안전)**.
→ `docs/specs/<과목>-curriculum.md` (영역별로). **범위 해설은 과설계를 막는 가드레일**이므로 꼭 포함(예: "깁스 자유 에너지 ✗", "적분 속도식 유도 ✗").

## 2단계 — 4종 교과서 페이지 인덱스
각 출판사 **학생 교과서 차례**를 읽어 단원/소단원 → 페이지를 매핑한다.
- 차례는 보통 **이미지** → `Read`(`pages` 범위; 앞쪽 4~9쪽 부근에 있는 경우가 많음). `pdftotext`는 차례 정렬이 깨지니 쓰지 말 것.
- **100MB↑ 교과서는 `Read` 불가** → 페이지만 분리 후 읽는다:
  ```bash
  pdfseparate -f <시작> -l <끝> "<교과서>.pdf" /tmp/p_%d.pdf
  pdfunite /tmp/p_*.pdf /tmp/toc.pdf     # 합본 → /tmp/toc.pdf 를 Read
  ```
- 출판사마다 소단원 묶음·표현·페이지가 다르므로 **성취기준을 축**으로 4종을 한 표에 매핑한다.
→ `docs/specs/<과목>-textbook-index.md` (학교 채택본 ★, 영역별 4종 페이지표, 활용법, 100MB 우회 메모).

## 3단계 — 마무리
- 이후 각 단원은 인덱스로 페이지를 찾아 **필요할 때 심층 분석**(4종 전체를 미리 분석하지 말 것 — 토큰·시간 낭비).
- 준비 끝 → [[lesson-blueprint]]로 단원 설계 시작.

## 함정 (이전 시행착오에서)
- ❌ 웹(나무위키 등)부터 검색 → ✅ **폴더의 공식 `[별책9]` 먼저**(웹은 403·부정확). 단, 공식본으로 **교차검증**해야 "확정".
- ❌ 교과서를 단원별로 쪼개기 → ✅ **전체 PDF + 페이지 범위 Read**.
- ❌ 차례를 `pdftotext`로 추출 → ✅ 차례는 **이미지라 Read**(텍스트는 순서가 깨짐).
- ❌ 100MB↑ 교과서를 그대로 Read → ✅ `pdfseparate`+`pdfunite`로 부분 추출.
