# flow-check — 거시 복귀 (전체 흐름 점검)

미시 제작([[slide-compose]])으로 한 장을 더한 뒤, **거시 관점으로 돌아와** blueprint와 지금까지의 슬라이드 묶음을 다시 본다. "슬라이드 하나하나"뿐 아니라 "전체적으로도 이상하지 않게" 만드는 단계다. [[chem-pedagogy-guidelines]] 기준.

## 언제
- 매 슬라이드 완성 직후(기본), 또는 사용자가 "흐름 봐줘"라고 할 때, 또는 단원 완성 시 1회 전체.

## 점검 항목
- **서사 아크 유지** — blueprint의 `도입→탐구→정리→형성평가` 흐름에서 이 슬라이드가 제자리인가? 갑자기 튀지 않는가?
- **연결** — 직전·직후 슬라이드와 매끄럽게 이어지나? 앞에서 안 깐 개념을 갑자기 쓰지 않나?
- **중복/공백** — 같은 내용을 두 번 다루거나, 목표에 필요한 단계가 빠지지 않았나?
- **인지부하·난이도 곡선** — 점점 쌓이게 배열됐나? 한 슬라이드가 과부하는 아닌가?
- **시간 배분** — 차시 분량에 맞나? 슬라이드 수가 과하거나 부족하지 않나?
- **시각 언어 일관성** — 흡열=빨강·발열=파랑, 태그·칩·표기가 단원 내내 일관적인가?
- **활동 유형 다양성** — 발문·시뮬·데이터·평가가 한쪽으로 쏠리지 않았나?

## 어긋나면 (거시 수정 → 미시 복귀)
1. 문제와 제안을 사용자에게 한두 줄로 제시(예: "3·4번이 같은 개념을 반복 → 4번을 응용 사례로 바꾸자", "도입과 탐구1 사이에 개념 다리 슬라이드 1장 추가 권장").
2. 합의되면 **`docs/specs/<lesson>-blueprint.md`의 슬라이드 개요 표를 수정**(삽입/삭제/재배열/의도 변경)하고 최종수정일 갱신.
3. 영향받는 슬라이드는 상태를 `planned`로 되돌리고 [[slide-compose]]로 다시 만든다(외과적 변경 — 무관한 슬라이드는 건드리지 않음).
4. 문제 없으면 "흐름 이상 없음"만 보고하고 다음 슬라이드로.

## 출력
- blueprint(있으면 갱신) + 사용자에게 짧은 점검 보고. 코드 산출물 없음.
