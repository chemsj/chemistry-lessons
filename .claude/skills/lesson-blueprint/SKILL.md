# 단원 blueprint — 거시 출발점

수업 제작은 **한 번에 슬라이드를 다 만들지 않는다.** 먼저 단원 전체의 **살아있는 설계도(blueprint)** 를 만든다. 이 문서가 이후 미시 제작([[slide-compose]])의 구동 목록이자, 흐름 점검([[flow-check]])의 기준이며, 진행 상태(중단·재개)를 담는다.

## 절차 (brainstorm: 한 항목씩 질문하며 좁힘)
사용자와 합의하며 아래를 채운다. 모르면 한 번에 다 묻지 말고 핵심부터 질문한다.
1. **단원 · 차시 · 대상** — 어느 단원, 몇 차시 분량, 학년/수준, 수업 시간.
2. **학습 목표** — 1~3개. 학생이 끝나고 무엇을 할 수 있어야 하는가.
3. **대상 오개념** — 이 주제에서 학생이 흔히 틀리는 지점(발문·형성평가의 표적).
4. **서사 아크** — `도입 질문 → 탐구(개념별) → 정리 → 형성평가`의 큰 흐름을 한 단락으로.
5. **슬라이드 개요 표** — 아크를 슬라이드 칸으로 분해. 각 칸은 **한 줄 의도 + 활동유형 + 레이아웃(제안)**. 활동유형은 [[lesson-activity-types]]에서 고른다. **레이아웃 칸은 AI 제안(잠정)일 뿐 — 최종 레이아웃은 [[slide-compose]] 단계에서 갤러리 후보를 제시해 사용자가 선택·확정한다.**

[[chem-pedagogy-guidelines]]를 항상 적용(탐구 우선, 실측 앵커, 표기는 학생이 배운 것만).

## 산출물: `docs/specs/<lesson-slug>-blueprint.md`
아래 틀로 저장한다. `<lesson-slug>`는 수업 폴더명(예 `chem-3-05-acid-base`).

```markdown
# <단원·차시 제목> — blueprint
- 폴더: <unit>/<NN-slug>/   · 대상: <학년> · 분량: <N>차시/<분>
- 상태: planning | building | done   · 최종수정: <YYYY-MM-DD>

## 학습 목표
1. …
## 대상 오개념
- …
## 서사 아크
도입 … → 탐구 … → 정리 … → 형성평가 …

## 슬라이드 개요
| # | 의도(한 줄) | 활동유형 | 레이아웃(제안) | 상태 |
|---|---|---|---|---|
| 1 | 도입 질문·동기 | — | single(제안) | planned |
| 2 | <개념> 탐구 | 입자 시뮬+발문 | 2col | planned |
| … | | | | |
| n | 마무리(마인드맵) | — | single | planned |

## 참고·출처
- 실측 앵커/수치(검증): …   (학생 화면 미언급 근거는 출처.md에)
```

상태값: `planned`(설계만) → `built`(슬라이드 생성) → `verified`(검증 통과). slide-compose가 한 장 만들 때마다 이 표의 상태를 갱신한다.

## 다음 단계
